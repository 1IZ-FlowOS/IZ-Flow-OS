# 🚀 IZ-Flow OS (Deutsch)

IZ-Flow ist ein experimentelles Betriebssystem-Konzept und eine eigene Programmiersprache, die Komplexität durch **funktionale Kürzel** ersetzt. Anstatt klassischer Syntax nutzt IZ-Flow **Logik-Ketten**, die direkt Systemressourcen, Grafiken und Prozesse steuern.

### 🛠 Die Syntax
In IZ-Flow wird alles durch **Kürzel** und **Hexadezimal-Werte** definiert. Das macht den Code extrem kompakt und hardwarenah.

**Beispiel:** `syp{iz{1f{dr{04{wt{05{L{1`
* **syp**: System-Prozess (Startet eine Funktion/Objekt)
* **iz{1f**: Information/Wert (Hex 1F = Dezimal 31)
* **dr{04**: Richtung (04 = Rechts)
* **wt{05**: Wait (Pause von 0,5 Sekunden)
* **L{1**: Lauf-Status (Aktiv)

### 🧱 Aktuelle Module
* **sy/syp**: Systemsteuerung & Prozesse
* **adm**: Admin-Verwaltung & Energie-Check
* **gf**: Grafik-Engine (Maps & Text)
* **st/fl**: Speicher-System (.iz Dateien)

# 🚀 IZ-Flow OS (English)

IZ-Flow is an experimental OS concept and a custom scripting language designed to replace syntax complexity with **functional shortcuts**. Instead of traditional code, IZ-Flow uses **data chains** to directly control system resources, graphics, and logic.

### 🛠 The Syntax
In IZ-Flow, everything is defined by **shorthand tags** and **hexadecimal values**. This keeps the code extremely compact and close to the logic level.

**Example:** `syp{iz{1f{dr{04{wt{05{L{1`
* **syp**: System Process (Starts a function/object)
* **iz{1f**: Information/Value (Hex 1F = Decimal 31)
* **dr{04**: Direction (04 = Right)
* **wt{05**: Wait (Pause for 0.5 seconds)
* **L{1**: Life Status (Active)

### 🧱 Current Modules
* **sy/syp**: System Control & Processes
* **adm**: Admin Management & Energy Monitoring
* **gf**: Graphics Engine (Maps & Text)
* **st/fl**: Storage System (.iz files)
